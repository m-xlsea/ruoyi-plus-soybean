## 分级

| Path | 策略 | 覆盖率 |
|:---|:---|:---|
| A | 改了什么手动验证 | 不强制 |
| B | 关键路径写测试, 边界场景 | ≥60% 变更代码 |
| C+ | 先写测试再实现, 全 TDD | ≥80% |

## 流程 (Path B+)

1. 从 plan.md 提取当前任务的验收标准
2. 先写测试 (或至少测试桩)
3. 实现代码
4. 运行测试, 失败→修复→再跑 (最多 3 轮)
5. 3 轮仍失败 → 标记"需要人工介入", cunzhi 通知

## 降级
测试框架不存在 → 先 `npm install` 安装, 或用断言脚本替代。
